Arc'kin Stack Overview

Systems, tools, and the source of truth.

1. The Philosophy: System Over Scatter

The Arc'kin Agency operates on the principle that standard AI usage is fundamentally scattered. Most workflows involve bouncing between disconnected tools and ephemeral chat sessions where every prompt starts from zero. This results in the loss of compounding value: insights are retyped, and decisions made on Monday are forgotten by Friday.

The Arc'kin system is the inverse of this scatter. It is a disciplined operating system designed to preserve intent and ensure that every action builds upon a documented foundation. This discipline is anchored by a single, non-negotiable rule regarding the relationship between the human operator and the machine.

Key Principle: The AI does not originate. The AI does not ideate without instruction, decide without scope, or architect without authorization. Every creative and strategic direction comes from the operator. The system carries the intent; the operator remains the boss.

This philosophy is enforced through a physical and digital rig that prioritizes execution and memory over generic chat interfaces.

2. The Architecture: The Team

The system relies on a named roster of ten specialist personas. Each operates within a specific lane to prevent the “drift” common in general-purpose chat.

  • BOSS: Michael Landi. Field Operator. Sets direction.
  • CONSIGLIERE (ORIENT): Briefing, coordination, and routing. The first point of contact for any ask.
  • STRATEGY LANE:
    • STRATEGOS: Revenue architecture, GTM, and commercial shape.
    • RECON: Research, intelligence, and market pulls.
  • BUILD LANE:
    • CANVAS: Creative direction, brand, and visual systems.
    • SIGNAL: Copy, voice, narrative, and content.
    • FORGE: Web development and technical implementation.
    • STUDIO: 3D, audio, motion, and media production.
  • OPS LANE:
    • CIRCUIT: Automations, integrations, and workflow plumbing.
  • LIFE LANE:
    • ANCHOR: Personal pillars and work-life integration.
  • ENGAGEMENT STEWARD:
    • ATLAS: Account memory, promise ledger, and correspondence continuity for specific engagements.

Work typically runs as a “swarm.” For example, a positioning question pulls STRATEGOS and RECON together, while a landing page build engages CANVAS, SIGNAL, and FORGE in parallel. ORIENT manages the routing to ensure the right specialist addresses the right question.

The primary execution surface is built on the Claude Max session engine. This is accessed through “three doors” that all lead to the same underlying vault.

The Operator's Advantage

A terminal-based execution surface like Claude Code fundamentally changes the relationship with the file system. Instead of the operator acting as a manual bridge — copying and pasting text — the system operates directly on the files. This allows for the simultaneous spawning of specialist agents to work on different parts of a project within a single, unified environment. While these tools execute, they do not “remember” without a dedicated layer.

4. The Memory Layer: Obsidian as the Source of Truth

The “Source of Truth” is the vault where all agency memory resides. The system utilizes Obsidian for this purpose, prioritizing local storage and plain Markdown files over proprietary cloud formats that risk deprecation.

This layer is load-bearing for three reasons:

  1. Continuity: When a new session begins, the relevant files are already the context. There is no “upload ritual” required to re-introduce the operator or the project.
  2. Token Economy: By maintaining SOPs and decision logs, the operator pays for new “signal” rather than repeating existing information.
  3. Organizational Sanity: The vault uses MOCs (Maps of Content) as a memory index. If a decision is not documented in the vault with a date, it did not happen.

The vault is a working surface designed for legibility under load. It prevents the system from starting every session from zero.

5. High-Bandwidth Capture and Infrastructure

To feed the memory layer efficiently, the stack utilizes specialized hardware to eliminate the friction of manual typing.

  • Plaud: A dedicated voice capture device. Voice is the highest-bandwidth input humans produce; Plaud turns that bandwidth into operable text without a typing layer.
  • Physical Rig: The system runs on a Mac Mini M1 (primary desktop) and a MacBook Pro (mobile mirror). This separation preserves the discipline between a deep-work environment and a travel surface.
  • Storage Philosophy: The rig is SSD-only (G-Tech G-RAID and SanDisk), prioritizing reliability and speed. No spinning disks are used in the pipeline.
  • Display Rig: A 49" Samsung ultrawide allows for the parallel windowing required for agent swarms — the desk physically mirrors the system diagram.

6. The Specialized Roster: Capability Over Popularity

The stack includes a roster of “sidekick” and “build” tools selected for specific technical roles rather than market trends.

Research Specialists

  • Perplexity (via Comet browser): Used for fast, open-web research with transparent sourcing.
  • NotebookLM: A second-brain reader used for closed-world synthesis across a bounded corpus of documents.
  • Gemini (AI Pro): Utilized for its extended context window and Workspace-native integrations.

Creative Surfaces

  • Figma & Figma Weave: The environment for design systems, interfaces, and AI-native motion and canvas work.
  • Flow (powered by Veo): Google's specialized surface for AI filmmaking and video production.
  • Stitch: Specialized AI UI/UX generation.
  • Gemini Canvas: Interactive data visualization for professional contexts.
  • Spline & Remotion: Tools for 3D scenes and programmatic, code-level video control.

Workflow Plumbing

  • Make, n8n, & Zapier: Automation and routing tools that move data between surfaces.
  • Jira: Used for project management and the epic-to-task decomposition of complex engagements.

7. Synthesis: How the Stack Moves

The value of the stack is realized in the “pass” — the sequence of moving from an idea to a finished artifact. This workflow is governed by documented standards and human authorization.

The Path of a Pass

  1. Ideation (Chat): Initial shaping and refining of the brief occur in standard Claude chat.
  2. Spec (Obsidian): Once refined, the specification is written into the Obsidian vault.
  3. Strategy Swarm: A Claude Code session opens. ORIENT routes the spec to STRATEGOS and RECON to produce a formal execution brief.
  4. Execution Swarm: A new session opens on that brief. Build agents (CANVAS, SIGNAL, FORGE, STUDIO) work in parallel to create the artifact.
  5. Taste Gates: The operator reviews the output against documented SOPs, scorecards, and style gates.
  6. Ship: The final product is delivered.

This sequence ensures that strategy always precedes execution and that the creative direction remains anchored to the operator's intent.

8. Closing Posture: Foundations for the Operator

Wave 1 is about establishing foundations. The rig and the software are not ends in themselves; they are the infrastructure for a repeatable and disciplined operating system. By separating ideation from implementation and ensuring each session is scoped to a specific beat, the system prevents drift.

Wave 1 is foundations. Wave 2 runs the same system at higher resolution.

The system is the leverage. The operator stays the operator.
