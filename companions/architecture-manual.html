Systems Architecture Manual

The Arc'kin Agency ecosystem.

1. The Foundational Philosophy: Intent vs. Execution

The Arc'kin philosophy is defined by the rigorous separation of the originator from the execution layer. AI is an engine for building, but it lacks the direction required for professional stakes. Fragmented tools are the default state for most users; a unified operating system is the goal. By isolating human intent from machine execution, the agency creates a structural safeguard against the entropy of generic AI outputs.

The objective is a move from the scattered state to the operating system state.

DimensionScattered StateOperating System State
ContextEvery session starts from zero; context is lost.A persistent vault provides continuous context.
ContinuityInsights are fragmented and temporary.Decision logs and SOPs ensure insights compound.
AgencyAI ideates without scope, leading to drift.The human operator maintains a monopoly on origination.

Origination remains a human-monopolized function. When AI dictates the architecture, output moves toward the mean of training data, sacrificing the strategic edge of a senior operator. The system prevents quality slip and chat drift by enforcing a single rule: the AI does not ideate without instruction or decide without scope. The system carries the intent; the operator remains the operator.

3. The Tool Stack: A Functional Taxonomy

Tools are not a buy-list; they are a collection of seats earned through utility. The stack is grouped by mental models rather than popularity. Roughly eighty percent of the work runs through the core execution engine.

Strategic Utility Pairings. The “High-Bandwidth Input” problem is solved by the integration of three components. Claude Max and Claude Code provide terminal-level execution. Obsidian serves as the sovereign source of truth. Plaud captures high-bandwidth voice input and converts it into operable text.

Functional Taxonomy.

  • Core: Claude Max — the session engine hosting Claude Code and Claude Cowork.
  • Memory: Obsidian — the sovereign vault for agents, SOPs, and decisions.
  • Capture: Plaud — hardware voice-to-text for meetings and thoughts.
  • Sidekick LLMs: Gemini AI Pro (extended context), NotebookLM (synthesis), Perplexity (research).
  • IDE / Launcher: Google Antigravity — the surface that launches Claude Code sessions.
  • Build & Design: Figma, Figma Weave, Make, Framer, Spline, Stitch, Gemini Canvas, Flow (Veo), Remotion.
  • Ops: n8n, Zapier, Jira.
  • Browser: Claude for Chrome, Comet.

4. The Workflow Logic: Routing, Swarming, and Taste Gates

The discipline of sequence dictates that strategy must precede execution and ideation must remain isolated from implementation.

Input Channels

  • Local: terminal prompts, native dictation, and file drops.
  • Remote: Claude Dispatch for quick retrievals and Claude Remote-control for session management.
  • Exterior: automations and triggers from the calendar, inbox, or vault events.

Claude Surfaces

The system operates through three doors to the same vault. Claude Code (via Antigravity) is the execution environment that reads and writes files directly. Claude Cowork is a project-scoped chat for shared context. Claude Chat is utilized for early shaping and brief refinement.

The Standard Pass

  1. Ideation: shaping occurs in Claude Chat to refine the brief.
  2. Documentation: the refined spec is committed to the Obsidian vault.
  3. Routing: ORIENT analyzes the spec and routes the task.
  4. The Swarm: agents collaborate across lanes. A strategy swarm (STRATEGOS and RECON) produces the brief, followed by an execution swarm (CANVAS, SIGNAL, FORGE, STUDIO) to build the artifact.
  5. The Taste Gate: this is the critical intervention point. The operator verifies the output against the original intent. The Taste Gate ensures execution meets the human standard before shipping.

5. The Ecosystem Map: Visual Synthesis

The Ecosystem Map is the cognitive anchor for the operating system. It serves as a visual grammar, allowing a senior operator to diagnose system bottlenecks at a glance.

The map utilizes directional flow lines to track work from input to artifact. Tool groups are represented by distinct node shapes, and pulsing lines indicate active swarms between collaborating agents.

  • Color Lanes: correspond to Strategy, Build, Ops, and Life lanes.
  • Node Shapes: represent specific tool groups (Core, Memory, Capture).
  • Dotted Lines: indicate memory and context flows from the vault.
  • Solid Lines: represent active work handoffs.
  • Pulsed Edges: highlight active swarming between agents.

6. The Memory Layer: Obsidian as the Sovereign Source of Truth

Plain text sovereignty is a matter of institutional sanity. Obsidian ensures the agency's memory is not locked in proprietary cloud formats. The vault is a portable, local-first source of truth.

Vault Structure

  • Personas: files for each specialist agent.
  • SOPs: documented standards and style gates.
  • MOCs: Maps of Content indexing the agency memory.
  • Sessions: date-stamped notes for every handoff.

The vault enables a functional token economy. By using Claude Code to read and write files directly, the system eliminates the upload ritual. Relevant files are already context. The vault provides continuity across sessions, reduces token waste, and maintains a clear record of authorized decisions.

8. Posture and Cadence: The Path Forward

The Arc'kin Agency operates on a wave cadence. Wave 1 establishes the foundations. Higher-resolution complexity is only introduced once these structural logics are solidified.

The posture is strictly operator-to-operator. There is no call to action or subscription ask. The value of a functional, disciplined system is self-evident to a senior practitioner. The system provides the leverage, but the operator stays the operator.
